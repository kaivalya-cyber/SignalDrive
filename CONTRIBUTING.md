# Contributing

## Reporting Bugs
Open an issue with:
- Your OS and Python version
- Steps to reproduce
- Terminal output / error message

## Suggesting Features
Open an issue tagged `enhancement` with a clear description of what you want and why.

## Submitting Code
1. Fork the repo
2. Create a branch: `git checkout -b feature/your-feature`
3. Commit your changes
4. Open a pull request against `main`

## Code Style
- Follow existing file structure — one responsibility per file
- All constants go in `config.py`, no magic numbers elsewhere
- Every function needs a docstring
